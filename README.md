# SNS Reaction (SillyTavern 확장)

채팅 메시지마다 **세계관 속 인물들의 SNS 반응**을 별도로 생성해서 보여주는 확장입니다.
스토리를 진행시키지 않고, 최근 대화 상황에 대한 트위터/인스타/페북/카톡/디스코드 반응만 뽑아줍니다.

- 지원 플랫폼: **Twitter(X), Instagram, Facebook, KakaoTalk, Discord**
- 플랫폼별 성격을 담은 **영어 기본 지침** 내장 + 사용자 수정/프리셋 저장 가능
- **{{user}} 사칭 포함/제외** 선택 가능 (메시지별 체크박스 + 전역 기본값)
- 생성 결과는 채팅 파일(`extra.sns_reactions`)에 저장되어 채팅을 다시 열어도 유지됨
- 라이트/다크 테마

대상 버전: SillyTavern **v1.18.x** (구버전 API 시그니처 폴백 포함)

---

## 설치 (폴더에 직접 넣기)

`sns-reaction` 폴더를 통째로 아래 위치에 복사하세요. 둘 중 하나면 됩니다.

**A. 현재 사용자 전용 (권장)**
```
SillyTavern/data/default-user/extensions/sns-reaction/
```
(다중 계정을 쓴다면 `default-user` 대신 본인 계정 폴더)

**B. 전체 사용자 공용**
```
SillyTavern/public/scripts/extensions/third-party/sns-reaction/
```

폴더 안에 `manifest.json`, `index.js`, `style.css`가 바로 있어야 합니다
(즉 `.../sns-reaction/manifest.json` 경로가 되도록).

복사 후 SillyTavern을 **완전히 새로고침**(Ctrl+F5)하면
확장 메뉴(🧩) → 설정 목록에 **"SNS 반응"** 드로어가 생깁니다.

---

## 사용법

1. 확장 설정에서 **SNS 반응 활성화** 체크 (기본 켜짐)
2. 채팅방의 각 메시지 아래에 `▸ SNS 반응` 토글이 생깁니다. 클릭하면 패널이 열립니다.
3. 원하는 플랫폼 버튼(Twitter / Instagram / Facebook / KakaoTalk / Discord)을 누르면
   해당 메시지 시점까지의 최근 대화(설정한 컨텍스트 수만큼)를 바탕으로 반응이 생성됩니다.
4. `{{user}} 사칭 포함` 체크박스로 유저 본인 명의 게시물/카톡 허용 여부를 그때그때 정할 수 있습니다.
5. 생성된 카드 우측 상단 🗑로 개별 삭제할 수 있습니다.

## 설정 항목

| 항목 | 설명 |
|---|---|
| 테마 모드 | 생성된 SNS 카드의 라이트/다크 |
| 컨텍스트 메시지 | 생성 시 참고할 최근 메시지 수 (기본 5) |
| 프로바이더 | **현재 연결 사용** = 지금 채팅에 쓰는 메인 API 그대로 사용. 그 외(OpenAI/Claude/Google AI Studio/Vertex AI/OpenRouter/Custom)는 SillyTavern **API 연결 탭에 저장해 둔 키**로 직접 호출 |
| 모델 | 직접 호출 시 사용할 모델명 (예: `gemini-3.1-pro-preview`) |
| {{user}} 사칭 기본 허용 | 메시지별 체크박스의 기본값 |
| 지시사항 프리셋 (플랫폼별) | 플랫폼 탭 선택 → 지시사항 수정 → 이름 입력 후 **저장(현재)** 하면 프리셋으로 저장 + 즉시 적용. 이름 없이 누르면 적용만 됨 |
| 저장 (전체) | 화면의 모든 설정을 한 번에 저장 |
| 공통 지침 | STRICT GENERATION MODE 지시문. 스토리 진행 금지 / JSON 형식 강제 등 |

## 프리셋 동작 방식

- 각 플랫폼은 항상 하나의 **활성 지시사항**을 가지며, 생성 시 그것이 사용됩니다.
- `-- New Preset --` 상태에서 텍스트를 고치면 활성 지시사항이 바로 바뀝니다.
- 프리셋을 선택하면 그 내용이 텍스트칸에 로드되고, **저장(현재)** 를 눌러야 활성 지시사항으로 적용됩니다.
- "이 플랫폼 기본값 불러오기"로 언제든 내장 기본 지침으로 되돌릴 수 있습니다.

## 문제 해결

- **아무것도 안 뜸**: F12 콘솔에서 `[SNS Reaction]` 로그/에러 확인. 폴더 경로와 새로고침(Ctrl+F5) 확인.
- **직접 프로바이더 호출 실패**: 해당 프로바이더의 API 키가 SillyTavern API 연결 탭에 저장되어 있어야 합니다. 안 되면 프로바이더를 "현재 연결 사용"으로 바꿔보세요.
- **JSON 파싱 실패 경고**: 모델이 형식을 어긴 경우로, 원문이 그대로 표시됩니다. 다시 생성하거나 공통 지침을 강화하세요.

## v1.1 추가 기능

- **API 키 직접 입력**: 프로바이더를 "현재 연결"이 아닌 것으로 선택하면 키 입력 칸이 나타납니다. 키를 입력하면 브라우저에서 해당 프로바이더로 직접 호출하고, 비워두면 SillyTavern API 연결 탭에 저장된 키로 서버 경유 호출합니다. (Vertex AI는 직접 입력 미지원 → ST 저장 인증 사용) ⚠️ 키는 settings.json에 평문 저장되므로 공용 PC에서는 주의.
- **출력 언어**: 자동(채팅 언어) / 한국어 / English / 日本語 / 中文 / 커스텀. 각 반응 카드의 🌐 버튼으로 이미 생성된 반응을 설정 언어(자동이면 한국어)로 번역할 수도 있습니다.
- **반응 길이**: 짧게 / 보통 / 길게 / 커스텀(자유 지시문).
- **캐릭터 정보 반영 / 월드 인포 반영 토글**: 세계관(로어북)에 맞는 이름·문화가 나오도록 컨텍스트 포함 여부를 제어합니다. 공통 지침에도 "배경에 맞는 이름 사용" 규칙이 추가되었습니다.
- **📌 스토리 반영**: 각 반응 카드의 📌 버튼으로 고정하면, 고정된 반응(최근 5개)이 이후 롤플레잉 프롬프트에 삽입되어 캐릭터가 SNS 내용을 알게 됩니다. 설정의 마스터 토글로 전체 켜기/끄기 가능. 기본적으로는 아무것도 주입되지 않습니다(핀을 꽂아야만 반영).
